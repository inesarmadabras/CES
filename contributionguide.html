Sobre Missão



Por favor leia o seguinte guia antes de publicar um issue ou enviar um pull request. Leia tambem o Código de Conduta antes de contribuir ou participar nas discussões.

Issues

  • Pedidos / Feature Requests têm que ser descritivos e devem conter informação sobre implementação
  • Bug reports devem descrever o problema em detalhe bem como o que o despoletou e, se possível, conter uma possivel solução.
  • Questões devem apenas ser sobre o funcionamento do código e sobre o respectivo projeto

Pull Requests

  • Solicitões de recursos (Feature requests) têm que ser discutidas e aceites através de um issue antes de enviar um pull request.
  • Correcções de Bugs (Bug fixes) devem conter regression tests.
  • Todos os pull requests devem seguir as guidelines do código.
  • Pull requests irão ser merged depois de ser revistos pelos moderadores
  • Por favor seja respeitador com os outros contribuidores e siga o Código de Conduta

Coding Standards

  • É boa prática escrever testes para a sua contribuição
  • Deverá consultar as guidelines completas aqui
